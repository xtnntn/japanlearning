# MVP 实现审计

> 审计依据：`docs/产品决策记录.md`  
> 更新日期：2026-07-11

当前试验包：`日语阅读日报_0.1.0_aarch64.dmg`，SHA-256 `9994ac5f48686e21fe2acdf32e7b49df2c59e912762ff7e2e37b2662c157c87fbf`。该包已包含能力画像、主动深入解释和严格两周实验状态。

## 已实现并有代码证据

- 每日唯一文章、次日过期且不形成补读债务；
- KAI-YOU 单源抓取、未读链接排除、正文长度门槛、会员/用户投稿过滤；
- 干净阅读器、原始图片、第三方视频或社媒来源、原文回链；
- 任意划词、日语提示优先、按需展开中文、振假名字段；
- 每日理解题持久化、逐字原文证据校验、答题结果记录；
- 初始定位、每周无辅助独立阅读评估及独立存储；
- 标题投票、题材反馈、利用/探索选文排序；
- 最近 14 天按文章长度和难度校正的划词趋势；
- 新语境旧表达与理解题绑定的独立理解率；
- Base URL、API Key、模型发现、Responses / Chat Completions 协议选择和 macOS Keychain 持久化；
- AI 设置弹窗使用独立滚动容器，避免滚轮穿透到阅读器；当 Rust keyring 无法读取旧 macOS Keychain 条目时，应用会安全回退到系统 `security` 查询同一条目，不输出或保存密钥明文。
- 用户主动安装、修改或移除的 `launchd` 每日标题提醒。
- release `.app` 与 `.dmg` 构建成功；App 完整 ad-hoc 签名通过 `codesign --verify --deep --strict`，DMG 通过 `hdiutil verify`；
- release App 实际启动后进程路径来自包内可执行文件，`--daily-reminder` 分支可独立运行并正常退出。
- macOS Keychain `apple-native` 后端通过临时凭证写入、读取、删除往返测试，测试不接触真实 API Key。
- KAI-YOU 解析已拆为纯 HTML 解析层，固定样本覆盖公开编辑部长文、受限/非编辑部页面、短文、日期、图片和视频/社媒嵌入；默认忽略的实时测试已验证当前首页候选仍存在兼容长文。
- 基础能力画像可查看初始定位、日常题、独立周测、中文展开率与完成量；用户可校正未来文章目标难度，系统自动建议和历史证据保持不变。
- 划词第一屏保持短提示；“深入解释”仅在用户主动点击后单独调用，并限制为最多三个相关语法/搭配点。
- 两周实验状态只使用完成文章，显示观察日、有效阅读日及三项验收的数据充分性；未满 14 天或样本不足时明确拒绝给出有效/无效结论。

## 已实现但仍需真实环境验证

- KAI-YOU 页面结构未来变化时仍需定期运行实时冒烟测试；
- 第三方 Chat Completions 网关对 JSON Schema 输出的兼容性；App 按 `json_schema`、`json_object`、纯 JSON 提示三层降级。
- 初始定位实际耗时是否接近 15–20 分钟；
- 每周评估文章在长期使用中是否始终未读且足够长。

## 两周后才能验收

- 归一化划词频率是否下降；
- 每周无翻译理解题成绩是否不退步；
- 新语境旧表达独立理解率是否上升；
- AI 题目覆盖历史表达的频率是否足以形成稳定指标。

App 内判定门槛：至少观察 14 天、完成 10 个阅读日、归一化划词趋势前后各至少 3 篇、至少 2 次独立周测、旧表达独立理解前后各至少 4 次验证。门槛用于防止用极少样本提前宣称有效。

## 尚未完成

## 明确不属于当前 MVP

- 内置 Anki 卡片与间隔复习；
- AI 复现句；
- 多站点内容源；
- 追番、音频转写和实时语音。
